JSOND

Introduction

JSOND (JSON Definition) is a way of defining JSON text using JSON text.

The purpose of JSOND is to be able to articulate JSON text without example values. JSOND is a superset of JSON so JSOND text have the same structure as the JSON text defined.

Basics

Let's recap JSON basics. JSON is built on two structures:

  • A unordered set of name:value pairs (object)
  • An ordered list of values (array)

In JSON, a value must be one of the following:

  • object
  • array
  • string
  • number
  • true
  • false
  • null

Values

JSOND defines four literals a special meaning:

  • "boolean"
  • "string"
  • "number"
  • "integer"

"boolean"

A JSOND value "boolean" defines that the JSON value must be either true or false.

"string"

A JSOND value "string" defines that the JSON value must be a string.

It is possible to further define the string using a colon followed by a regular expression.

E.g. "string:^(male|female)$" defines that the JSON value must be either "male" or "female".

"number"

A JSOND value "number" defines that the JSON value must be a number.

It is possible to further define the number using a colon followed by a interval notation.

E.g. "number:[1,4)" defines that the number must be equal to or larger than 1 and lower than 4.

"integer"

A JSOND value "integer" defines that the JSON value must be an integer.

It is possible to further define the integer using a colon followed by a interval notation.

E.g. "integer:[1,4)" defines that the number must be either 1, 2 or 3.

Names

undefined

A JSOND name may be followed by a comma and the string undefined to express that the JSON name:value pair may be absent from the JSON text.

null

A JSOND name may be followed by a comma and the string null to express that the JSON value may be null.
