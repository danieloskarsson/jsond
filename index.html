JSOND

Introduction

JSOND (JSON Definition) is a fat-free way of defining JSON using JSON.

The purpose of JSOND is to be able to articulate JSON text without example values. JSOND is a superset of JSON so JSOND text have the same structure as the JSON text defined.

JSON Basics

Let's recap JSON basics. JSON is built on two structures:

  • An ordered list of values (array)
  • A unordered set of name/value pairs (object)

In JSON, a value must be one of the following:

  • object
  • array
  • string
  • number
  • true
  • false
  • null

JSOND Values

When used as a value, JSOND defines four string literals with a special meaning.

boolean

A JSOND value "boolean" defines that the JSON value must be either true or false.

string

A JSOND value "string" defines that the JSON value must be a string.

The string can be further defined using a colon followed by a regular expression.

"string:^(red|green|blue)$" defines that the JSON value must be either "red", "green" or "blue".

number

A JSOND value "number" defines that the JSON value must be a number.

The number can be further defined using a colon followed by a (mathematical) set or interval.

"number:{1.5,2,4.8}" defines that the JSON value must be 1.5, 2.4, or 4.8.

"number:[1,4)" defines that the JSON value must be a number greater than or equal to 1, and less than 4.

integer

A JSOND value "integer" defines that the JSON value must be an integer,
i.e. a number without a decimal component.

The integer can be further defined using a colon followed by a (mathematical) set or interval.

"integer:{1,2,4}" defines that the JSON value must be 1, 2, or 4.

"integer:[1,4)" defines that the JSON value must be an integer greater than or equal to 1, and less than 4.

JSOND References

JSOND text can be persisted as a .jsond file. A JSOND file can be referenced using the http or file URI scheme as the JSOND value.

"http://jsond.org/utc.jsond" defines that the JSON value must be a string with the format "yyyy-mm-ddThh:mm:ssZ" with optional fractions of the second.

JSOND Names

When preceeded by a colon and used as the end of a name, JSOND defines two string literals with a special meaning.

null

A JSOND name that end with a colon followed by the string null defines that in addition to the value defined, the JSON value may be null.

"name:null" defines that the JSON value of may be null.

undefined

A JSOND name that end with a colon followed by the string undefined defines that the JSON name/value pair is not a mandatory part of the JSON text.

"name:undefined" defines that the JSON name/value pair may not be in the JSON text.

"name:null|undefined" or "name:undefined|null" defines that the JSON value may be null and that the JSON name/value pair may not be in the JSON text.

A. Examples

Example 1 defines a person with a required first and last name and a optional age that if given must have a value that is an positive integer. 
{
    "firstName": "string",
    "lastName": "string",
    "age:undefined": "integer:[0,"
}

EXAMPLE 1: PERSON

Example 2 defines a product with a required id, name and price and a optional array of strings named tags. 
{
    "id": "integer",
    "name": "string",
    "price": "number:(0",
    "tags:undefined": [
        "string"
    ]
}

EXAMPLE 2: PRODUCT

JSOND is designed to be a minimal language and does not provide a way to require unique items or a certain number of items in a array.

Example 3 defines a array of products. A product is defines with a required id, name and price, a optional array of strings named tags, a optional dimensions object and a optional warehouse location. The definition of a warehouse location is found in the refrenced file geo.jsond. 
[
    {
        "id": "integer",
        "name": "string",
        "price": "number:(0",
        "tags:undefined": [
            "string"
        ],
        "dimensions:undefined": {
            "length": "number",
            "width": "number",
            "height": "number"
        },
        "warehouseLocation:undefined": "http://jsond.org/geo.jsond"
    }
]
